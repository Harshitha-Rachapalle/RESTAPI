Imagine you (the client) are sitting at home and ordering food using Swiggy/Zomato.

Behind the scenes, everything works using the **HTTP protocol**.

## What is HTTP?

HTTP (HyperText Transfer Protocol) is a communication language between:

Client ‚Üí Browser / Mobile App / Postman

Server ‚Üí Backend system

It defines:

How requests are sent

How responses are returned

The format of data

**HTTP is stateless**

The server forgets you the moment the response is sent.

# HTTP REQUEST STRUCTURE

When a client talks to a server, it sends an HTTP Request.

URI

Method

Headers

Body

## 1.URI (Address of the Resource)

 **Story**: Home Address

To visit a friend, you need:

House number

Street

City

Without address ‚Üí you can‚Äôt reach.

### What is URI?

URI **(Uniform Resource Identifier)** is the address of a resource on the server.

```
https://api.amazon.com/products/101
```

`products` ‚Üí resource

`101` ‚Üí specific product

## 2. Method (What do you want to do?)

 **Story**: Talking to Shopkeeper 

‚ÄúShow me this item‚Äù ‚Üí GET

‚ÄúAdd this item‚Äù ‚Üí POST

‚ÄúUpdate quantity‚Äù ‚Üí PUT

‚ÄúRemove item‚Äù ‚Üí DELETE

| Method | Meaning         |
| ------ | --------------- |
| GET    | Fetch data      |
| POST   | Create new data |
| PUT    | Replace data    |
| PATCH  | Partial update  |
| DELETE | Remove data     |

```
GET /users/1
```

Get user with ID 1

## 3.Headers (Extra instructions)

**Story**: Courier Instructions 

Fragile

Urgent

Handle with care

These are headers.

### What are Headers?

Headers provide **metadata** about the request.

```
Content-Type: application/json
Authorization: Bearer token123
Accept: application/json
```

 Meaning:

Who you are
What format you speak
Security info

## 4. Body (Actual data ‚Äî optional)

**Story**: Letter Content 

The actual message you write inside envelope.

üîπ Meaning

Body contains the data sent to the server.

**Example (Create User)**
```
POST /users
```

```
{
  "name": "John",
  "email": "john@test.com"
}
```

 GET requests usually do not have a body.

 # HTTP RESPONSE STRUCTURE

Body

| ---- | ------------ |

| 400  | Bad request  |

| 403  | Forbidden    |

# HTTP METHODS (CRUD MAPPING)

CRUD = Create, Read, Update, Delete

### GET ‚Äî Read

```
GET /orders/789
```

 **Used for:**
 
Fetch data

No body

Safe & Idempotent

### POST (CREATE)

**Story**: Filling Admission Form 

Creates new record.

```
POST /orders

{
  "productId": 101,
  "quantity": 2
}

```

 **Used to:**

Place orders
Register users
Upload files

üìå Not idempotent (repeating creates duplicates)

### PUT (FULL UPDATE)

**Story**: Replacing Old Phone 

Old phone ‚Üí new phone

**example:**
```
PUT /users/101


{
  "name": "Harshitha",
  "email": "h@example.com"
}
```

Replaces entire resource

Idempotent

### PATCH (PARTIAL UPDATE)

**Story**: Changing Only Mobile Number

**example:**

```
PATCH /users/101


{
  "email": "new@example.com"
}
```

Partial update

 Efficient

 ### DELETE (REMOVE)
 
 **Story**: Deleting Old Account 

 ```
DELETE /orders/789
```

Removes resource

Idempotent

### HEAD

**Story**: Checking Parcel Status 

You ask:

‚ÄúHas it arrived?‚Äù

Don‚Äôt want the parcel yet

**Meaning**

Same as GET

But no response body

Only headers

**example:**

```
HEAD /products/123
```

Used for:

Health checks
Size validation
Caching logic

### OPTIONS

**Story**: Asking Store Rules 

‚ÄúCan I return items?‚Äù

üîπ Meaning

Tells which methods are allowed.

‚ÄúWhat actions are allowed here?‚Äù

**example:**

```

```

